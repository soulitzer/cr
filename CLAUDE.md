# Code Review PRs

When asked to create a review/analysis PR in https://github.com/soulitzer/cr/:
- PR title matches the source PR title
- PR body contains only a link to the source PR
- File is plain text (no markdown formatting)
- File is named <question>.txt where <question> is a snake_case
  question summarizing what the doc analyzes
- Content should be readable as plain text without rendering — use
  indentation and whitespace for structure instead of markdown syntax
  like #, ```, |---|, or emoji
